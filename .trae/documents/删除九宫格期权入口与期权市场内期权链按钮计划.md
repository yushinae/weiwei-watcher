## Proposed Changes（改动方案）
> 只做“删除/清理”，不改视觉风格与顶部悬浮菜单（OptionsDropdown）的卡面与行为。

### A) `src/App.tsx`：九宫格菜单移除期权分类与期权条目
**目标**：九宫格菜单不再出现“期权”分类；“所有”列表也不再出现“期权市场/期权链”两项与跳转。

#### A1. 收窄 `activeCat` 联合类型（删除 `'options'`）
当前：`'all'|'options'|'data'|'account'|'help'`  
改为：`'all'|'data'|'account'|'help'`

#### A2. 左侧分类数组移除“期权”
删除 `{ k: 'options' as const, l: '期权' }`。

#### A3. 右侧 items 映射移除期权项并删除 `options:` 分支
- 在 `map.all` 中删除两项：
  - `{ label: '期权市场', ... to:'/options' }`
  - `{ label: '期权链', ... to:'/options-chain' }`
- 删除 `map.options` 整个 key（因为分类已不存在）。
- 同步保证 `Record<typeof activeCat, ...>` 的 key 与 `activeCat` 类型一致，避免 TS 报错或运行时取不到映射。

#### A4. 清理无用 import（如 `LineChart`、`Layers`）
如果删除后 `LineChart` / `Layers` 在整个 `App.tsx` 不再使用，则从 lucide import 中移除，保持 lint 通过。

---

### B) `src/pages/OptionsMarketBrowser.tsx`：删除顶部“期权链”按钮与跳转
**目标**：期权市场弹层 header 里不再出现“期权链”按钮，不再通过该按钮跳转到 `/options-chain`；只保留顶部 OptionsDropdown 的期权链入口。

#### B1. 删除 header 右侧“期权链”按钮 JSX
移除：
```tsx
<button onClick={handleOpenChain}>... 期权链</button>
```
并确保 header 右侧仍只保留必要动作（如关闭按钮），间距 `gap` 合理。

#### B2. 删除 `handleOpenChain` 及其依赖
- 删除 `handleOpenChain` 回调函数。
- 若 `DetailOverlay` 内不再需要 `useNavigate()` 的 `navigate` 变量，则一并删除该变量（但注意文件其他区域可能仍用到 `useNavigate`，不要误删 import）。

#### B3. 清理 `ExternalLink` 等未使用 import
移除 lucide 的 `ExternalLink`（如果不再使用），保证 TS/构建无 warning。
